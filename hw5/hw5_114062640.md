**5. In implementing the features, we see variables like lanes, tl_rsc, tl_name, tl_device, bitmap, iface, etc., used to store different Layer's protocol information. Please explain what information each of them stores.**
bitmap:以Bit形式記錄所有可用或被允許的傳輸資源。
tl_rsc:傳輸層資源 (Transport Layer Resource)。描述硬體原始能力的結構，包含該資源屬於哪個 Domain、是什麼類型。
tl_name:transport layer的名稱，例如ud_verbs、tcp或shm。
tl_device:儲存物理硬體裝置的名稱，例如 mlx5_0或 eth0。
iface:介面 (Interface)。當 Worker打開一個 tl_rsc 後，它就變成一個通訊物件 iface。
lanes:傳輸最終選擇的路徑。UCP 會將多個 iface 組合。例如：Lane0用於小訊息，Lane1用於大量資料傳輸。作業Line 2就是輸出這邊的資訊。
